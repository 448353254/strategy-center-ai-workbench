# 策略中心 AI 工作台 MVP 精简版 PRD

## 1. 产品定位

策略中心 AI 工作台是一套面向游戏营销策略部门的内部效率工具。

MVP 阶段不追求一次性覆盖完整策略业务，而是优先解决三个高频痛点：

1. 历史资料找不到、复用难。
2. 客户 Brief 解构和方案大纲产出耗时。
3. 项目排期、任务进度、延期风险缺少统一管理。

## 2. MVP 目标

### 2.1 业务目标

- 建立一个可上传、可检索、可复用的策略资料库。
- 支持基于 Brief 和资料库内容生成需求解构、QA 清单和方案大纲。
- 支持策略项目的排期管理、任务跟进和延期预警。

### 2.2 不做什么

MVP 阶段明确不做以下能力：

- 不做实时热点抓取。
- 不做实时舆情监测。
- 不做全自动精美 PPT 生成。
- 不做答疑模拟陪练。
- 不做复杂投标胜率预测。
- 不做多人在线协同编辑。
- 不做完整 ERP/OA 替代。

## 3. 用户角色

| 角色 | 权限定位 | 典型使用场景 |
| --- | --- | --- |
| 策略负责人 | 管理项目、查看团队资料和风险 | 查看项目进度、分配任务、检查延期风险 |
| 策略成员 | 使用资料、生成方案、更新任务 | 搜资料、解析 Brief、生成大纲、更新进度 |
| 协作人员 | 查看授权项目、上传协作资料 | 上传媒介/商务/设计资料，更新协作任务 |
| 管理员 | 管理用户、权限、标签和系统配置 | 配置角色权限、维护资料分类 |

## 4. MVP 功能范围

MVP 只包含 3 个一级模块：

| 模块 | 核心价值 | 优先级 |
| --- | --- | --- |
| 资料库 | 解决资料沉淀、检索、复用问题 | P0 |
| 方案助手 | 解决 Brief 解构和方案大纲产出问题 | P0 |
| 项目跟进 | 解决项目排期、任务进度、风险预警问题 | P0 |

### 6.2 页面内容

- 我的待办任务

- 进行中项目
- 最近上传资料

### 6.3 快捷入口

## 7. 模块一：资料库

## 7.1 功能目标

建立策略部门统一资料沉淀入口，支持上传、解析、分类、检索和复用。

## 7.2 文件上传

### 支持文件类型

- PPT/PPTX
- Word/DOCX
- PDF
- Excel/XLSX
- TXT/Markdown
- 图片 JPG/PNG

### 上传字段

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 文件 | 文件 | 是 | 支持单个或批量上传 |
| 资料名称 | 文本 | 是 | 默认取文件名，可修改 |
| 资料类型 | 单选 | 是 | 方案、PPT模板、文案、报告、讲稿、复盘、竞品资料等 |
| 所属项目 | 关联 | 否 | 可关联到具体项目 |
| 游戏类型 | 多选 | 否 | MMORPG、SLG、二次元、女性向、竞技、休闲等 |
| 项目类型 | 多选 | 否 | 新品上线、版本更新、活动推广、品牌联动等 |
| 营销节点 | 多选 | 否 | 预热、上线、长线运营、节日活动等 |
| 可见范围 | 单选 | 是 | 仅自己、项目成员、策略部门、指定人员 |
| 敏感等级 | 单选 | 是 | 普通、内部、保密 |

## 7.3 文档解析

上传后系统异步解析文档。

解析内容：

- 文件基础信息
- 正文文本
- 表格文本
- PPT 页标题和正文
- 图片 OCR 文本
- 文档摘要
- 关键词
- 自动标签

解析状态：

- 待解析
- 解析中
- 解析成功
- 解析失败

## 7.4 资料检索

支持两种检索方式：

1. 关键词检索。
2. 自然语言检索。

自然语言检索示例：

- “找一个适合二次元手游新品上线的方案。”
- “有没有春节活动相关文案？”
- “找最近一年中标项目用过的 PPT 结构。”
- “找适合女性向游戏的舆情风险参考。”

检索结果展示：

| 字段 | 说明 |
| --- | --- |
| 资料名称 | 文件或资产名称 |
| 资料类型 | 方案、文案、报告等 |
| 匹配度 | 高、中、低或百分比 |
| 匹配原因 | AI 说明为什么推荐 |
| 可复用点 | AI 提炼可直接参考的内容 |
| 操作 | 查看、下载、引用到项目 |

## 7.5 AI 复用建议

用户打开资料后，可点击“生成复用建议”。

AI 输出：

- 资料核心摘要
- 适合复用的场景
- 可直接借鉴的结构/文案/策略点
- 当前项目使用时需要修改的地方
- 不建议复用的风险点

## 7.6 资料库验收标准

- 能上传并保存文件。
- 能解析 PPT、Word、PDF、Excel 的主要文本内容。
- 能生成摘要、关键词和标签。
- 能按关键词和筛选条件检索资料。
- 能通过自然语言检索返回相关资料。
- 能展示 AI 匹配原因和复用建议。
- 权限受限资料不能被无权限用户查看或下载。

## 8. 模块二：方案助手

## 8.1 功能目标

帮助策略成员从客户 Brief 快速生成需求解构、客户确认问题和方案大纲。

## 8.2 Brief 解析

### 输入内容

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 项目名称 | 文本 | 是 | 本次方案项目名称 |
| 游戏名称 | 文本 | 是 | 对应游戏名称 |
| 项目类型 | 单选 | 是 | 新品上线、版本更新、活动推广、品牌联动等 |
| 客户 Brief | 文件/文本 | 是 | 可上传文件或粘贴文本 |
| 沟通记录 | 文件/文本 | 否 | 客户会议纪要、聊天记录等 |
| 禁忌要求 | 文本 | 否 | 不可提及、不可使用的方向 |
| 方案用途 | 单选 | 是 | 投标、客户提案、内部汇报 |

### AI 输出

生成《需求解构报告》，包含：

- 项目背景摘要
- 客户核心需求
- 客户次要需求
- 隐性需求推测
- 关键约束条件
- 需求优先级
- 方案风险点
- 需向客户确认的问题清单

### 输出要求

- 每一条需求需标注来源：来自 Brief、沟通记录或 AI 推测。
- AI 推测内容必须明确标注“需人工确认”。
- 支持一键复制和导出 Markdown/Word。

## 8.3 方案大纲生成

### 输入内容

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 需求解构报告 | 关联 | 是 | 来自 Brief 解析结果 |
| 方案调性 | 单选 | 是 | 专业严谨、热血、有趣、年轻化、国风等 |
| 页数范围 | 数字 | 否 | 例如 15-25 页 |
| 必含模块 | 多选 | 是 | 背景、洞察、策略、创意、执行、预算、排期等 |
| 参考资料 | 关联资料 | 否 | 从资料库选择 |
| 客户偏好 | 文本 | 否 | 客户过往偏好或注意事项 |

### AI 输出

生成《方案大纲》，包含：

- 方案标题建议
- 整体叙事逻辑
- PPT 目录
- 每页标题
- 每页核心内容
- 每页需要补充的数据或素材
- 可引用的资料库内容

### 推荐大纲结构

| 页段 | 内容 |
| --- | --- |
| 1 | 封面 |
| 2 | 项目背景 |
| 3-4 | 市场与用户洞察 |
| 5-6 | 竞品与机会分析 |
| 7-9 | 核心策略 |
| 10-12 | 创意与传播玩法 |
| 13-15 | 执行规划 |
| 16 | 项目排期 |
| 17 | 资源与预算 |
| 18 | 效果预估 |
| 19 | 风险与应对 |
| 20 | 团队优势/结尾 |

## 8.4 内容检查

MVP 支持文本级检查，不做复杂视觉审美检查。

输入：

- 方案大纲
- PPT 文案初稿
- 客户 Brief
- 检查维度

检查维度：

- 是否覆盖客户核心需求
- 逻辑是否连贯
- 模块是否完整
- 是否存在明显空泛表达
- 是否触碰禁忌要求
- 是否缺少数据或案例支撑

AI 输出：

| 字段 | 说明 |
| --- | --- |
| 问题位置 | 页码、模块或段落 |
| 问题等级 | 严重、一般、建议 |
| 问题说明 | 具体问题 |
| 修改建议 | 如何修改 |
| 示例改写 | 可选 |

## 8.5 方案助手验收标准

- 能上传或粘贴 Brief。
- 能生成结构清晰的需求解构报告。
- 能生成客户确认 QA 清单。
- 能基于需求解构生成方案大纲。
- 能从资料库推荐相关参考资料。
- 能对方案初稿输出问题清单和修改建议。
- AI 输出支持编辑、复制、导出。

## 9. 模块三：项目跟进

## 9.1 功能目标

帮助策略负责人和成员管理项目进度、任务责任人、关键节点和延期风险。

## 9.2 项目列表

### 项目字段

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 项目名称 | 文本 | 是 | 项目唯一名称 |
| 游戏名称 | 文本 | 是 | 对应游戏 |
| 客户名称 | 文本 | 否 | 客户主体 |
| 项目类型 | 单选 | 是 | 新品上线、版本更新、活动推广、品牌联动等 |
| 项目负责人 | 用户 | 是 | 策略负责人或主策划 |
| 项目成员 | 多用户 | 否 | 参与成员 |
| 启动日期 | 日期 | 是 | 项目开始日期 |
| 方案提交日期 | 日期 | 否 | 关键节点 |
| 讲标日期 | 日期 | 否 | 关键节点 |
| 执行日期 | 日期 | 否 | 关键节点 |
| 项目状态 | 单选 | 是 | 未开始、进行中、已完成、已暂停、已归档 |
| 风险等级 | 单选 | 自动 | 正常、一般、紧急、严重 |

## 9.3 项目排期表

创建项目后，可选择“AI 生成排期”。

输入：

- 项目类型
- 总周期
- 关键节点
- 不可调整节点
- 人员资源限制
- 客户截止时间

AI 输出标准排期表：

| 字段 | 说明 |
| --- | --- |
| 环节 | 调研、策略、创意、方案、内部评审、客户提交、讲标等 |
| 任务名称 | 具体任务 |
| 负责人 | 可为空，由用户确认 |
| 协作部门 | 商务、媒介、设计、投放等 |
| 开始日期 | 计划开始 |
| 截止日期 | 计划完成 |
| 前置依赖 | 依赖任务 |
| 状态 | 未开始、进行中、已完成、延期 |
| 延期原因 | 用户填写 |
| 风险等级 | AI 自动判断 |

用户可手动调整排期。

## 9.4 甘特图

基于项目排期表生成甘特图。

展示内容：

- 任务时间跨度
- 任务负责人
- 任务状态
- 关键节点
- 延期任务标红

支持视图：

- 按日
- 按周
- 按月

## 9.5 进度更新

任务负责人可更新：

- 完成状态
- 当前进度百分比
- 实际完成日期
- 延期原因
- 备注

状态枚举：

- 未开始
- 进行中
- 已完成
- 延期
- 暂停

## 9.6 延期预警

AI 根据任务截止时间、完成状态、关键节点影响判断风险。

风险等级：

| 等级 | 判断逻辑 |
| --- | --- |
| 正常 | 当前任务按计划推进 |
| 一般 | 单个非关键任务轻微延期 |
| 紧急 | 延期可能影响方案提交、讲标或执行 |
| 严重 | 已影响不可调整节点或核心交付 |

AI 输出：

- 延期任务
- 影响范围
- 风险等级
- 可能后果
- 调整建议

调整建议示例：

- 压缩后续非关键环节时长。
- 增加协作人员分担文案或资料整理。
- 提前安排内部评审，避免客户提交前集中修改。
- 与客户确认是否可调整非核心交付内容。

## 9.7 人员负载看板

MVP 做基础版，不做复杂产能预测。

展示字段：

- 成员姓名
- 当前负责项目数
- 当前任务数
- 本周到期任务数
- 已延期任务数
- 负载状态

负载状态规则：

- 0-2 个任务：偏低
- 3-5 个任务：正常
- 6-8 个任务：偏高
- 9 个及以上任务：过载

## 9.8 项目跟进验收标准

- 能创建、编辑、归档项目。
- 能维护项目关键节点。
- 能生成和编辑项目排期表。
- 能基于排期表展示甘特图。
- 能更新任务状态和延期原因。
- 能自动识别延期任务并生成风险等级。
- 能展示人员基础负载。

## 15. MVP 成功指标

建议试点 4-6 周后评估。

| 指标 | 目标 |
| --- | --- |
| 资料上传量 | 不低于 200 份历史资料 |
| 资料检索使用率 | 每周至少 50 次检索 |
| Brief 解析采纳率 | 60% 以上结果被编辑后继续使用 |
| 方案大纲生成使用率 | 每周至少 5 个真实项目使用 |
| 项目排期覆盖率 | 80% 以上进行中项目录入系统 |
| 延期预警准确率 | 负责人主观认可率 70% 以上 |
| 用户满意度 | 试点用户评分不低于 7/10 |

## 16. 一期实施建议

### 16.1 推荐周期

MVP 建议 8-10 周完成。

### 16.2 阶段拆分

## 17. 待确认问题

正式进入研发前，需要确认：

- 是否私有化部署。
- 是否允许使用外部大模型 API。
- 历史资料目前存放在哪里。
- 是否已有统一用户体系或需要单独账号。
- PPT/PDF 是否大量为图片扫描件。
- 资料权限是否按项目隔离。
- 是否需要对接现有 OA、ERP 或企业微信。
- MVP 是否要求导出 Word/PDF。
- 甘特图是否需要导出图片或 Excel。
